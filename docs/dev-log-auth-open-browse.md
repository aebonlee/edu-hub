# DreamIT Edu Hub — 사이트 자유 열람 + 링크만 로그인 필수

## 개발일지

**작성일:** 2026-03-18
**작업 유형:** 접근 제어 정책 변경
**상태:** 완료

---

## 변경 사유

기존에는 모든 페이지에 AuthGuard를 적용하여 로그인하지 않으면 어떤 페이지도 볼 수 없었음.
사이트 내용을 전혀 확인할 수 없는 상태에서 로그인을 강제하는 것은 사용자 경험상 부적절.

## 변경 내용

### 자유 열람 (로그인 불필요)

| 페이지 | 경로 |
|--------|------|
| 홈 (메인) | `/` |
| 학습사이트 목록/상세 | `/courses`, `/courses/:id` |
| Edu Hub 소개 | `/about` |
| 학습사이트 분양문의 | `/franchise` |
| 공지사항 목록/상세 | `/notice`, `/notice/:id` |
| Q&A 목록/상세 | `/qna`, `/qna/:id` |
| 스토어 | `/shop` |

### 로그인 필수 (개인 활동)

| 페이지 | 경로 | 사유 |
|--------|------|------|
| 공지 작성 | `/notice/write` | 관리자 글쓰기 |
| Q&A 작성 | `/qna/write` | 질문 작성 |
| 마이페이지 | `/mypage` | 개인 정보 |
| 주문 이력 | `/mypage/orders` | 개인 정보 |
| 장바구니 | `/cart` | 결제 활동 |
| 결제 | `/checkout` | 결제 활동 |
| 주문 확인 | `/order-confirmation` | 결제 결과 |

### 학습사이트 외부 링크

학습사이트 카드의 "사이트 방문하기" 버튼은 기존 로그인 게이트(`Courses.jsx` 내부)가 처리.
로그인하지 않은 상태에서 링크 클릭 시 로그인 안내 표시.

## 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `src/layouts/PublicLayout.jsx` | 열람 페이지에서 AuthGuard 제거, 개인 활동 페이지만 AuthGuard 유지 |
